Projeto 1: Módulo 1 - Construindo Seu Primeiro Portfólio Web Pessoal

Este projeto marca o ponto alto do seu aprendizado no Módulo 1. Você aplicará todos os conceitos essenciais de HTML e CSS que exploramos até agora para criar uma versão inicial do seu próprio Portfólio Web Pessoal. Pense nele como sua primeira vitrine online, exibindo suas habilidades para o mundo!

Objetivo do Projeto

Seu principal objetivo aqui é demonstrar sua capacidade de estruturar uma página web com HTML e estilizá-la com CSS, focando em:

  • Uso correto da **estrutura básica do HTML**, incluindo `<!DOCTYPE html>`, `<html>`, `<head>` e `<body>`.
  • Aplicação de **elementos de texto** como `<p>` (parágrafo) e títulos `<h1>` a `<h6>`.
  • Criação de **listas** utilizando `<ul>` (não ordenada), `<ol>` (ordenada) e `<li>` (item de lista).
  • Inserção de **links** com a tag `<a>` e **imagens** com a tag `<img>`.
  • Compreensão e manipulação do **modelo de caixa**, utilizando propriedades CSS como `margin`, `padding` e `border`.
  • Utilização eficaz de **seletores CSS** por `tag`, `.classe` e `#ID` para aplicar estilos específicos.
  • Início da **responsividade básica** com a meta tag `viewport` (`<meta name="viewport" content="width=device-width, initial-scale=1.0">`) e a implementação de media queries simples para adaptar o layout a diferentes tamanhos de tela.

Requisitos Essenciais

Seu portfólio deve ser composto por, no mínimo, as seguintes seções e elementos:

  1. Página Inicial (index.html já existente ou adaptado)

    • Um **cabeçalho** (`<header>`) que contenha seu nome ou um título principal (`<h1>`).
    • Uma **barra de navegação** (`<nav>`) com links (`<a>`) para as principais seções:
      • Início (apontando para a própria página index.html)
      • Sobre Mim (apontando para sobre-mim.html, a página que você criou na atividade de revisão)
      • Contato (uma seção simples na mesma página inicial ou em uma página separada)
      • Módulo 1 (um link para o painel.html ou uma página de "Aulas" se preferir)
    • Uma **seção principal de boas-vindas** (`<main>` ou `<section>`) com um texto introdutório sobre você, usando a tag `<p>`.
    • Uma **imagem sua** ou uma imagem representativa do seu trabalho, inserida com a tag `<img>`.
    • Uma seção simples de **"Projetos em Destaque"** (pode ter 1 ou 2 cards bem básicos, com um título `<h3>` e uma imagem `<img>`).
    • Um **rodapé** (`<footer>`) com informações de copyright, usando a tag `<p>`.

  2. Página "Sobre Mim" (sobre-mim.html - da atividade de revisão)

    • Você deve adaptar e aprimorar a página "Sobre Mim" que desenvolveu na atividade de revisão das Aulas 1-4.
    • Esta página deve incluir: sua foto (`<img>`), uma descrição detalhada (`<p>`), uma lista de suas habilidades (organizada com `<ul>` e `<li>`), e links para redes sociais (usando `<a>`).

  3. Estilização CSS (style.css e/ou outros arquivos CSS)

    • Aplique **estilos consistentes** em todo o site (escolha fontes, cores e defina espaçamentos que criem uma identidade visual).
    • Utilize o **modelo de caixa** (`margin`, `padding`, `border`, `box-sizing`) de forma eficaz para controlar o espaçamento e as dimensões dos elementos.
    • Implemente **seletores de classe** (ex: `.nome-da-classe`) e **ID** (ex: `#nome-do-id`) de forma organizada e semântica no seu CSS.
    • Assegure que a **meta tag viewport** (`<meta name="viewport" content="width=device-width, initial-scale=1.0">`) esteja presente no `<head>` de todas as suas páginas para garantir que o layout se adapte a dispositivos móveis.
    • Adicione **transições** simples a elementos interativos (como links da navegação ou botões), utilizando a propriedade CSS `transition` para criar efeitos suaves.

Entrega do Projeto

Para entregar seu projeto finalizado, siga os passos abaixo:

  1. **Organização:** Reúna todos os seus arquivos (HTML, CSS e quaisquer pastas de imagens ou outros recursos) em uma única pasta, por exemplo, meu-portfolio-modulo1.
  2. **GitHub:** Faça o upload completo desta pasta para um repositório que você criar no **GitHub**.
  3. **Publicação:** Publique seu site online utilizando o serviço gratuito **GitHub Pages** (lembre-se da Aula 12!).
  4. **Compartilhamento:** Compartilhe o **link público** do seu site. Este será o URL que outras pessoas poderão usar para acessar seu portfólio.
  5. (Opcional, mas recomendado): Adicione um pequeno arquivo README.md na raiz do seu repositório GitHub. Neste arquivo, explique brevemente o projeto, quais tecnologias foram usadas e quaisquer desafios ou aprendizados significativos.

Lembre-se que a **qualidade do seu código** (aplicação de boas práticas) e a **organização dos arquivos** no repositório serão aspectos importantes da avaliação. Boa sorte e divirta-se criando!
