Boas Práticas em HTML e CSS

Escrever código limpo, organizado e eficiente não é apenas para os outros lerem; é também para você mesmo no futuro! Boas práticas tornam o código mais fácil de manter, escalar e depurar.

1. Estrutura do HTML: Semântica e Acessibilidade

Use Tags Semânticas

Em vez de usar apenas `<div>` e `<span>` para tudo, utilize tags HTML5 que descrevem o conteúdo: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`. Isso melhora a acessibilidade e o SEO.


<!-- Ruim -->
<div id="header">...</div>
<div class="navigation">...</div>

<!-- Bom -->
<header>...</header>
<nav>...</nav>

Atributos `alt` em Imagens

Sempre forneça descrições significativas nos atributos `alt` para imagens. Isso é crucial para acessibilidade (leitores de tela) e SEO.


<!-- Ruim -->
<img src="foto.jpg">

<!-- Bom -->
<img src="foto-do-por-do-sol.jpg" alt="Pôr do sol vibrante sobre o oceano com nuvens laranjas">

Estrutura Lógica de Títulos

Utilize a hierarquia de títulos (`<h1>` a `<h6>`) de forma lógica e sequencial. Deve haver apenas um `<h1>` por página (o título principal).

2. Organização e Estilo do CSS

Use Classes Significativas

Nomeie suas classes de forma descritiva e consistente (ex: `header-main-nav`, `product-card-title`, `btn-primary`). Evite nomes genéricos (`div1`, `itemA`).

Organize seu CSS

Agrupe as propriedades CSS relacionadas e organize suas regras. Você pode organizar por:

  • Seção do site: `header`, `main`, `footer`, `sidebar`.
  • Componente: `button`, `card`, `form`.
  • Tipos de propriedade: `layout` (display, position), `box-model` (margin, padding, border), `typography`, `colors`.

Use comentários para seções maiores (ex: `/* --- Seção de Navegação --- */`).

Evite Estilos Inline e `!important`

Estilos inline (`<div style="...">`) são difíceis de manter. O uso de `!important` quebra a cascata do CSS e deve ser evitado ao máximo.

Comentários e Indentação

Use comentários para explicar código complexo ou decisões de design. Mantenha uma indentação consistente para facilitar a leitura.

Unidades Relativas para Responsividade

Prefira `rem`, `em`, `%`, `vw`, `vh` em vez de pixels fixos para tipografia e dimensões, tornando seu design mais responsivo e escalável.

3. Validação de Código

Sempre valide seu HTML e CSS para encontrar erros e garantir que seu código esteja de acordo com os padrões da web.

  • Validador HTML: W3C Markup Validation Service
  • Validador CSS: W3C CSS Validation Service

Publicando seu Site: Colocando-o Online!

Depois de criar e testar seu site localmente, o próximo passo é torná-lo acessível para o mundo! Isso envolve hospedar seus arquivos em um servidor.

Passos para Publicar um Site Estático:

  1. Organize seus Arquivos

    Certifique-se de que todos os seus arquivos (HTML, CSS, imagens, etc.) estão na estrutura de pastas correta e que todos os caminhos (`href`, `src`) estão corretos (relativos ou absolutos).

    
meu-projeto/
├── index.html
├── painel.html
├── aula1.html
├── aula2.html
├── ...
├── aula12.html
├── style.css
└── imagens/
    ├── logo.png
    └── background.jpg

  2. Escolha uma Plataforma de Hospedagem Gratuita (para começar)

    Para projetos simples de HTML/CSS como este, existem ótimas opções gratuitas:

    • GitHub Pages: Excelente para projetos que já estão no GitHub. Você cria um repositório, faz o upload dos arquivos, e o GitHub Pages serve seu site diretamente do repositório.
      • Como fazer:
        1. Crie uma conta no GitHub.
        2. Crie um novo repositório (ex: `meu-portfolio`).
        3. Faça o upload de todos os seus arquivos para este repositório.
        4. Vá para as "Settings" (Configurações) do repositório, encontre a seção "Pages".
        5. Escolha a branch `main` (ou `master`) e a pasta `/(root)` ou `/docs` se você tiver uma pasta `docs`.
        6. Salve. Em alguns minutos, seu site estará acessível em um URL como `[seu-usuario].github.io/[nome-do-repositorio]`.
    • Netlify: Outra opção popular e muito fácil de usar. Permite arrastar e soltar sua pasta de projeto ou conectar-se diretamente ao GitHub. Oferece implantação contínua e HTTPS.
      • Como fazer:
        1. Crie uma conta no Netlify.
        2. No dashboard, clique em "Add new site" (Adicionar novo site) > "Deploy manually" (Implantação manual).
        3. Arraste e solte sua pasta de projeto (a pasta `curso-html-css` completa, por exemplo) para a área indicada.
        4. O Netlify irá implantar seu site e fornecer um URL automaticamente.
    • Vercel: Similar ao Netlify, com uma excelente experiência para desenvolvedores e integração fácil com repositórios Git.
      • Como fazer: Siga um processo similar ao Netlify, conectando-se ao GitHub ou fazendo o upload da pasta.

    Escolha a que você se sentir mais confortável. Para este curso, **GitHub Pages** é uma excelente escolha, pois também o familiariza com o controle de versão.

  3. Teste seu Site Online

    Após a publicação, acesse seu site pelo URL fornecido e teste-o em diferentes navegadores e dispositivos (use as ferramentas de desenvolvedor do navegador para simular dispositivos móveis).
