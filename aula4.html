Aula 4: HTML Semântico e Formulários - Dando Significado e Interatividade

Nesta aula, você aprofundará seus conhecimentos em HTML aprendendo sobre HTML Semântico, que melhora a estrutura e o significado do seu conteúdo, e sobre Formulários, que permitem coletar informações dos usuários e criar interatividade.

1. HTML Semântico: Construindo com Significado

O HTML Semântico consiste em usar as tags HTML de acordo com o significado do conteúdo que elas representam, e não apenas por sua aparência visual (que é trabalho do CSS!). Isso é crucial para:

  • Acessibilidade: Leitores de tela e outras tecnologias assistivas entendem melhor a estrutura da página.
  • SEO (Search Engine Optimization): Motores de busca (Google, Bing) compreendem melhor o seu conteúdo, o que pode melhorar seu ranqueamento.
  • Manutenibilidade: Seu código fica mais claro e fácil de entender e manter por outros desenvolvedores (e por você mesmo no futuro!).

Tags Semânticas Comuns:

  • <header>: Representa o cabeçalho de um documento ou de uma seção, geralmente contendo o logotipo, título principal e navegação.
  • <nav>: Contém links de navegação (menus).
  • <main>: Representa o conteúdo principal do documento. Deve haver apenas uma tag <main> por página.
  • <section>: Representa uma seção genérica de conteúdo (ex: "Sobre Mim", "Projetos"). Geralmente, tem um cabeçalho (<h1> a <h6>) como filho.
  • <article>: Representa um conteúdo independente e auto-contido, como um post de blog, um comentário ou uma notícia.
  • <aside>: Conteúdo relacionado, mas separado do conteúdo principal (ex: barras laterais, citações).
  • <footer>: Representa o rodapé de um documento ou seção, geralmente contendo informações de direitos autorais, links de contato, etc.
  • <figure> e <figcaption>: Usadas para agrupar uma imagem, vídeo, diagrama, etc., com sua legenda.
  • <time>: Representa uma data e/ou hora.

Exemplo de Estrutura Semântica:


<body>
    <header>
        <h1>Título do Site</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">Sobre</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section>
            <h2>Minha História</h2>
            <p>Conteúdo sobre minha história...</p>
        </section>

        <article>
            <h2>Artigo Recente</h2>
            <p>Este é um artigo sobre...</p>
            <footer>Publicado em <time datetime="2024-07-25">25 de julho de 2024</time>.</footer>
        </article>
    </main>

    <aside>
        <h3>Publicidade</h3>
        <p>Anúncio aqui.</p>
    </aside>

    <footer>
        <p>© 2024 Meu Site.</p>
    </footer>
</body>

Note que o seu arquivo `index.html` já utiliza muitas dessas tags semânticas para organizar o portfólio!

2. Formulários HTML: Coletando Dados do Usuário

Os formulários são usados para coletar informações do usuário, como nomes, e-mails, senhas, mensagens, etc. Eles são essenciais para login, contato, pesquisas e muito mais.

2.1. A Tag <form>: O Contêiner do Formulário

Todos os elementos de um formulário devem estar dentro da tag <form>. Ela possui atributos importantes:

  • action: O URL para onde os dados do formulário serão enviados após o envio.
  • method: O método HTTP usado para enviar os dados. Os mais comuns são:
    • GET: Os dados são anexados ao URL (visíveis na barra de endereço). Ideal para buscas ou dados não sensíveis.
    • POST: Os dados são enviados no corpo da requisição (não visíveis no URL). Ideal para dados sensíveis (senhas) ou grandes volumes de dados.

<form action="/enviar-contato" method="post">
    <!-- Elementos do formulário aqui -->
</form>

2.2. Elementos de Entrada (<input>)

A tag <input> é a mais versátil. Seu comportamento muda com o atributo type.

  • type="text": Para entrada de texto curto (nomes, endereços). 
    <label for="nome">Nome:</label>
<input type="text" id="nome" name="nome" placeholder="Seu nome completo">
  • type="email": Para endereços de e-mail. Inclui validação básica de formato. 
    <label for="email">E-mail:</label>
<input type="email" id="email" name="email" required>
  • type="password": Para senhas (esconde o texto digitado). 
    <label for="senha">Senha:</label>
<input type="password" id="senha" name="senha">
  • type="number": Para números. 
    <label for="idade">Idade:</label>
<input type="number" id="idade" name="idade" min="18" max="99">
  • type="date": Para datas. 
    <label for="data">Data de Nascimento:</label>
<input type="date" id="data" name="nascimento">
  • type="radio": Para seleção de uma única opção em um grupo. 
    
<input type="radio" id="opcao1" name="grupo" value="valor1"> <label for="opcao1">Opção 1</label><br>
<input type="radio" id="opcao2" name="grupo" value="valor2"> <label for="opcao2">Opção 2</label>

    Importante: Rádios com o mesmo `name` formam um grupo onde apenas um pode ser selecionado.

  • type="checkbox": Para seleção de múltiplas opções. 
    
<input type="checkbox" id="item1" name="hobbies" value="ler"> <label for="item1">Ler</label><br>
<input type="checkbox" id="item2" name="hobbies" value="esportes"> <label for="item2">Esportes</label>
  • type="submit": Cria um botão para enviar o formulário. 
    <input type="submit" value="Enviar Formulário">

2.3. Outros Elementos de Formulário:

  • <textarea>: Para entrada de texto longo (mensagens, comentários). 
    <label for="mensagem">Sua Mensagem:</label>
<textarea id="mensagem" name="mensagem" rows="5" cols="30"></textarea>
  • <select> e <option>: Para listas suspensas (dropdowns). 
    
<label for="cidade">Selecione sua cidade:</label>
<select id="cidade" name="cidade">
    <option value="sp">São Paulo</option>
    <option value="rj">Rio de Janeiro</option>
    <option value="bh" selected>Belo Horizonte</option> <!-- 'selected' pré-seleciona -->
</select>
  • <label>: Associa um texto descritivo a um controle de formulário. **Essencial para acessibilidade!** O atributo `for` do `<label>` deve ser igual ao `id` do input que ele descreve. 
    <label for="meuInput">Meu Campo de Texto:</label>
<input type="text" id="meuInput" name="campo">
  • <button>: Cria um botão genérico. O `type` padrão é `submit`, mas pode ser `button` (para JavaScript) ou `reset`. 
    <button type="submit">Enviar Dados</button>

Atributos Importantes para Inputs:

  • name: Nome do campo, usado para identificar o dado quando o formulário é enviado. **Obrigatório para que o dado seja enviado!**
  • id: Identificador único, usado para associar com `<label>` e para estilização/JavaScript.
  • value: O valor inicial do campo ou o valor que será enviado para `radio`, `checkbox`, `select`.
  • placeholder: Texto de dica que aparece no campo antes de o usuário digitar.
  • required: Faz com que o campo seja obrigatório (o formulário não será enviado se estiver vazio).
  • min, max, step: Para `type="number"`, `range`, `date`, etc., definem limites e incrementos.
