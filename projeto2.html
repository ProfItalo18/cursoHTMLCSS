Projeto 2: Módulo 2 - Criando um Layout Responsivo de Cartões

Este projeto foca na sua habilidade de construir layouts complexos e responsivos usando as ferramentas mais poderosas do CSS. Você criará uma página web que exibe uma coleção de "cartões" (cards), como os que você vê em sites de produtos, notícias ou portfólios. O grande desafio será fazer com que esses cartões se organizem de forma fluida em diferentes tamanhos de tela.

Objetivo do Projeto

O objetivo principal é que você demonstre o domínio dos seguintes conceitos do Módulo 2:

  • Entendimento e aplicação avançada do **Modelo de Caixa** (`margin`, `padding`, `border`, `box-sizing`).
  • Uso do **Posicionamento Básico** (`static`, `relative`, `absolute`, `fixed`) quando necessário para elementos específicos.
  • Criação de layouts flexíveis e responsivos utilizando **Flexbox**.
  • (Opcional, mas altamente recomendado): Implementação de layouts complexos usando **CSS Grid**.
  • Domínio de **Media Queries** para criar breakpoints e adaptar o layout para dispositivos móveis, tablets e desktops.
  • Manter a **semântica do HTML** e a **organização do CSS**.

Requisitos Essenciais

Sua página de cartões deve conter:

  1. Estrutura Básica (`index.html` ou `cards.html`)

    • Uma página HTML com a estrutura básica (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`).
    • Um **cabeçalho** (`<header>`) com um título principal (`<h1>`) e uma barra de navegação simples (`<nav>`).
    • Uma **seção principal** (`<main>` ou `<section>`) que contenha o contêiner dos cartões.
    • Um **rodapé** (`<footer>`) com informações de copyright.

  2. Layout dos Cartões

    • Crie um mínimo de **4 a 6 cartões** (`<div class="card">` ou `<article class="card">`).
    • Cada cartão deve conter:
      • Uma imagem (`<img>`).
      • Um título (`<h3>`).
      • Um parágrafo de descrição (`<p>`).
      • Um botão ou link (`<button>` ou `<a>`).
    • Utilize **Flexbox** para organizar o conteúdo dentro de cada cartão (ex: imagem acima do texto, botão no final).
    • Utilize **Flexbox** ou **CSS Grid** para organizar os cartões dentro de um contêiner principal.

  3. Responsividade (o ponto crucial do projeto)

    • Em telas de **desktops grandes** (ex: acima de 1200px), os cartões devem ter 4 colunas.
    • Em telas de **desktops/laptops menores** (ex: entre 992px e 1199px), os cartões devem ter 3 colunas.
    • Em telas de **tablets** (ex: entre 768px e 991px), os cartões devem ter 2 colunas.
    • Em telas de **dispositivos móveis** (ex: abaixo de 767px), os cartões devem ter 1 coluna e ocupar quase toda a largura disponível.
    • Utilize **Media Queries** (`@media`) para implementar esses breakpoints e mudar o layout do grid/flexbox.

  4. Estilização e Interatividade

    • Aplique estilos visuais consistentes e atraentes (cores, fontes, sombras, bordas arredondadas para os cartões).
    • Adicione efeitos de `hover` nos cartões (ex: uma leve elevação com `transform: translateY()`, ou uma mudança de sombra).
    • Garanta que o `box-sizing: border-box;` esteja aplicado globalmente (ou nos elementos necessários) para facilitar o cálculo das dimensões.

Entrega do Projeto

Para entregar seu projeto finalizado, siga os passos abaixo:

  1. **Organização:** Reúna todos os seus arquivos (HTML, CSS e quaisquer pastas de imagens) em uma única pasta (ex: layout-cards-modulo2).
  2. **GitHub:** Faça o upload completo desta pasta para um repositório que você criar no **GitHub**.
  3. **Publicação:** Publique seu site online utilizando o serviço gratuito **GitHub Pages**.
  4. **Compartilhamento:** Compartilhe o **link público** do seu site.
  5. (Opcional, mas recomendado): Adicione um pequeno arquivo README.md na raiz do seu repositório GitHub explicando brevemente o projeto, as tecnologias usadas e os desafios enfrentados, especialmente em relação à responsividade.

A atenção aos detalhes na responsividade e na organização do seu código CSS será crucial para a avaliação deste projeto.
