Aula 10: Responsividade e Media Queries - Criando Sites Adaptáveis

A web moderna exige que os sites se adaptem a uma vasta gama de dispositivos, de celulares minúsculos a telas de desktop gigantescas. Isso é o que chamamos de Responsividade. A principal ferramenta do CSS para alcançar a responsividade são as Media Queries.

O Que é Responsividade?

Um design responsivo garante que a aparência e o layout do seu site se ajustem automaticamente ao tamanho da tela e às características do dispositivo do usuário. Isso significa:

  • **Layout Flexível:** Elementos se redimensionam ou reposicionam.
  • **Imagens Adaptáveis:** Imagens que não transbordam ou ficam muito grandes.
  • **Textos Legíveis:** Tamanhos de fonte adequados para cada tela.
  • **Navegação Otimizada:** Menus que se transformam (ex: hambúrguer menu em mobile).

O objetivo é oferecer a melhor experiência possível para o usuário, independentemente de como ele acessa seu site.

A Meta Tag Viewport: O Primeiro Passo Essencial

Antes de usar Media Queries, você precisa garantir que o navegador saiba como escalar seu conteúdo. Isso é feito com a meta tag `viewport` no `<head>` do seu HTML:


<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • width=device-width: Define a largura da viewport para a largura do dispositivo (em pixels CSS).
  • initial-scale=1.0: Define o nível de zoom inicial quando a página é carregada.

Sem esta tag, navegadores móveis tentarão renderizar a página como se fosse em um desktop, o que resultará em uma experiência ruim.

Media Queries: A Ferramenta Mágica

Media Queries permitem aplicar estilos CSS apenas quando certas condições são verdadeiras, como o tamanho da tela, a orientação (retrato/paisagem), ou o tipo de mídia (tela, impressão).

A sintaxe básica é:


@media tipo de mídia e (característica: valor) {
    /* Estilos CSS aplicados quando a condição é verdadeira */
}

Tipos de Mídia Comuns:

  • screen: Para telas de computador, tablets, smartphones, etc. (o mais comum).
  • print: Para impressoras.
  • all: Para todos os dispositivos.

Características de Mídia Comuns:

  • width / height: Largura/altura da viewport.
  • min-width: Mínima largura da viewport.
  • max-width: Máxima largura da viewport.
  • orientation: portrait (altura maior que largura) ou landscape (largura maior que altura).
  • resolution: Resolução do dispositivo (ex: min-resolution: 2dppx para telas Retina).

Exemplos de Media Queries na Prática:

1. Alterando Estilos para Telas Pequenas (Mobile First Approach)

A abordagem Mobile First é uma prática recomendada: você escreve o CSS para a menor tela primeiro, e depois usa Media Queries para adicionar estilos para telas maiores.


/* Estilos padrão (aplicados a todas as telas, mas otimizados para mobile) */
body {
    font-size: 16px;
}
.header {
    padding: 10px;
}
.menu {
    display: none; /* Esconde o menu principal em mobile */
}

/* Media Query para telas maiores que 600px (tablets e desktops) */
@media screen and (min-width: 601px) {
    body {
        font-size: 18px;
    }
    .header {
        padding: 20px;
    }
    .menu {
        display: block; /* Mostra o menu em telas maiores */
    }
}

Um exemplo clássico é alterar o número de colunas de um layout Grid:

Item Grid 2

(1 coluna em telas pequenas, 2 colunas em telas médias, 3 colunas em telas grandes)

Considerações Importantes:

  • **Mobile First vs. Desktop First:** Embora Mobile First seja geralmente preferido, Desktop First (`@media (max-width: Xpx)`) também funciona. Escolha a que faz mais sentido para seu projeto.
  • **Breakpoints:** São os pontos de interrupção onde você aplica as Media Queries. Não existem breakpoints "universais", mas alguns comuns são 480px, 768px, 992px, 1200px. Escolha breakpoints que façam sentido para o seu conteúdo.
  • **Unidades Relativas:** Use `em`, `rem`, `%`, `vw`, `vh` para tamanhos de fonte, larguras, etc., pois eles se ajustam melhor do que pixels fixos.
  • **Imagens Responsivas:** Use `max-width: 100%; height: auto;` em suas imagens para que elas se adaptem ao contêiner.

Situações-Problema: Desafios Práticos com Responsividade!

Aplique o que aprendeu para resolver os seguintes cenários. Crie um novo arquivo HTML temporário (ex: `desafios-aula10.html`) e um CSS associado para testar suas soluções.

Problema 1: Menu de Navegação Adaptativo

Você tem um menu de navegação (`<ul>` com `<li>` e `<a>`). Em telas grandes (acima de 768px), ele deve aparecer como uma linha horizontal. Em telas menores (até 768px), os itens devem ser empilhados verticalmente e ocupar a largura total.

HTML:


<nav class="main-nav">
    <ul>
        <li><a href="#">Item 1</a></li>
        <li><a href="#">Item 2</a></li>
        <li><a href="#">Item 3</a></li>
    </ul>
</nav>

CSS:


/* Insira seu código CSS aqui para resolver o Problema 1 */
/* Estilos para mobile first */
.main-nav ul {
    /* ... */
}
.main-nav li {
    /* ... */
}
/* Media Query para telas maiores */
@media screen and (min-width: 769px) {
    .main-nav ul {
        /* ... */
    }
    .main-nav li {
        /* ... */
    }
}

Problema 2: Redimensionamento de Imagem para Fundo

Você tem uma imagem que serve como fundo de uma seção (`div`). Ela deve cobrir toda a área da seção (`background-size: cover;`) em telas grandes. No entanto, em telas menores (até 480px), você quer que a imagem de fundo seja diferente e use `background-size: contain;` para que a imagem inteira seja visível.

HTML:


<div class="hero-section">
    <h2>Seção de Destaque</h2>
</div>

CSS:


/* Insira seu código CSS aqui para resolver o Problema 2 */
.hero-section {
    background-image: url('https://via.placeholder.com/1200x400?text=Fundo+Grande'); /* Imagem para telas grandes */
    min-height: 300px;
    background-repeat: no-repeat;
    background-position: center center;
    /* ... estilos para telas grandes */
}

@media screen and (max-width: 480px) {
    .hero-section {
        background-image: url('https://via.placeholder.com/600x300?text=Fundo+Pequeno'); /* Imagem para telas pequenas */
        /* ... estilos para telas pequenas */
    }
}

Problema 3: Mudança de Cores e Bordas

Crie uma caixa (`div`) com um parágrafo dentro. A caixa deve ter uma borda azul e um fundo branco por padrão. Em telas com orientação paisagem (`landscape`), a borda deve se tornar verde e o fundo cinza claro. Em telas muito pequenas (largura máxima de 400px), a borda deve sumir e o fundo ser vermelho claro.

HTML:


<div class="color-box">
    <p>Esta caixa muda de cor!</p>
</div>

CSS:


/* Insira seu código CSS aqui para resolver o Problema 3 */
.color-box {
    border: 2px solid blue;
    background-color: white;
    padding: 20px;
    margin: 20px;
}

@media screen and (orientation: landscape) {
    .color-box {
        /* ... */
    }
}

@media screen and (max-width: 400px) {
    .color-box {
        /* ... */
    }
}

Problema 4: Ocultar Elemento em Mobile

Você tem uma barra lateral (`<aside>`) que contém informações adicionais. Em telas grandes, ela deve ser visível. Em telas menores que 768px, a barra lateral deve ser completamente ocultada para otimizar o espaço na tela.

HTML:


<div style="display: flex;">
    <main style="flex: 1;">
        <h2>Conteúdo Principal</h2>
        <p>Este é o conteúdo principal da página.</p>
    </main>
    <aside class="sidebar">
        <h3>Barra Lateral</h3>
        <p>Informações extras.</p>
    </aside>
</div>

CSS:


/* Insira seu código CSS aqui para resolver o Problema 4 */
.sidebar {
    /* Estilos padrão para telas grandes */
    width: 250px;
    background-color: #f0f0f0;
    padding: 15px;
}

@media screen and (max-width: 768px) {
    .sidebar {
        /* ... */
    }
}
