Projeto 3: Módulo 3 - Desenvolvendo um Site Interativo e Otimizado

Este é o seu projeto final do curso de HTML e CSS! Você terá a oportunidade de criar um site completo, com múltiplas páginas ou seções, aplicando tudo o que aprendeu sobre estruturação, estilização avançada, responsividade, e adicionando toques de interatividade visual com transições e animações. O foco será na entrega de um produto de alta qualidade, otimizado e pronto para ser publicado.

Objetivo do Projeto

O objetivo principal é que você demonstre o domínio abrangente dos tópicos do Módulo 3 e a capacidade de integrar diferentes conhecimentos para criar um site funcional e esteticamente agradável, com foco em:

  • Consolidação da **Responsividade e Media Queries** para garantir que o site funcione perfeitamente em qualquer dispositivo.
  • Uso criativo de **Transições CSS** para criar interações suaves e elegantes.
  • Implementação de **Animações CSS** para adicionar dinamismo e chamar a atenção para elementos importantes.
  • Aplicação de **Boas Práticas de Codificação** (HTML semântico, CSS organizado e eficiente).
  • Experiência prática na **Publicação do seu site** utilizando o GitHub Pages, garantindo que ele esteja acessível online.

Requisitos Essenciais

Seu site deve conter, no mínimo, as seguintes características:

  1. Estrutura do Site (Múltiplas Páginas ou Seções)

    • Um site com **pelo menos 3 páginas HTML distintas** (ex: Página Inicial, Serviços/Produtos, Contato) ou uma **single-page** com 3 seções bem definidas (`<section>`) e navegação suave entre elas (utilizando links internos e rolagem suave).
    • Utilização de elementos HTML semânticos como `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>` e `<footer>`.
    • Um **formulário de contato** funcional (não precisa enviar dados para um servidor, apenas a estrutura com campos como `<input type="text">`, `<input type="email">`, `<textarea>` e `<button type="submit">`).

  2. Responsividade Avançada

    • O layout deve ser **totalmente responsivo**, adaptando-se a telas de smartphones, tablets e desktops de forma fluida.
    • Utilize **Media Queries** de forma estratégica para otimizar a experiência em diferentes tamanhos de tela (ex: alterando layouts de Flexbox/Grid, tamanhos de fonte, visibilidade de elementos).
    • Garanta que as imagens sejam responsivas (ex: `max-width: 100%; height: auto;`).

  3. Transições e Animações CSS

    • Implemente **pelo menos 3 transições CSS** em elementos interativos (ex: mudança de cor de fundo em botões ao `hover`, suavização em menus de navegação, transformação de elementos ao passar o mouse).
    • Crie **pelo menos 1 animação CSS** que seja visualmente interessante e funcional (ex: um loader simples, um elemento que entra na tela com um efeito ao carregar, um ícone pulsante). Utilize `@keyframes` para definir a animação.

  4. Boas Práticas e Otimização

    • **Código HTML limpo e bem identado.**
    • **CSS organizado:** utilize comentários, classes semânticas, e, se desejar, organize seu CSS em seções ou até mesmo em múltiplos arquivos (`<link rel="stylesheet" href="caminho/arquivo.css">`).
    • **Otimização de Imagens:** utilize imagens com tamanhos e formatos adequados para web para garantir um bom desempenho.
    • Acessibilidade básica (ex: uso de atributos `alt` em imagens, estrutura de cabeçalhos correta).

Entrega do Projeto

Para entregar seu projeto finalizado, siga os passos abaixo:

  1. **Organização:** Organize todos os seus arquivos (HTML, CSS, imagens, etc.) de forma lógica em uma pasta principal (ex: meu-site-final).
  2. **GitHub:** Faça o upload completo desta pasta para um **novo repositório** no seu perfil do **GitHub**.
  3. **Publicação:** Publique seu site online utilizando o **GitHub Pages**. Certifique-se de que o site está acessível publicamente.
  4. **Compartilhamento:** Forneça o **link público** do seu site para avaliação.
  5. (Obrigatório): Adicione um arquivo README.md na raiz do seu repositório GitHub. Este README deve incluir:
    • Uma breve descrição do seu projeto.
    • As tecnologias HTML e CSS utilizadas.
    • Quais funcionalidades de responsividade, transições e animações foram implementadas.
    • Quaisquer desafios que você enfrentou e como os superou.
    • Uma captura de tela ou GIF do seu site (opcional, mas recomendado).

Este projeto é a sua chance de mostrar tudo o que você aprendeu. Dedique-se à qualidade do código, à experiência do usuário e à apresentação final. Boa sorte!
